# metadata 字段透传问题分析与修复需求

> 发现日期：2026-06-09
> 发现过程：会话 ID 机制调研
> 涉及文件：src-tauri/src/modules/transform/claude_openai.rs

---

## 一、调研背景

用户询问 tauri-gateway 是否有会话 ID 机制，以及转发时是否会出错。为回答此问题，对比分析了三个项目的会话管理机制：

| 项目 | 类型 | 会话 ID 用途 |
|------|------|-------------|
| sub2api | 账号池代理 | 会话固定（绑定账号） |
| cc-switch | 本地代理 | 日志追踪、会话恢复 |
| tauri-gateway | 端点转发 | 当前无显式处理 |

---

## 二、调研过程

### 2.1 第一步：分析 tauri-gateway 的头部透传逻辑

文件：`src-tauri/src/modules/proxy/forward.rs:501-519`

```rust
for (k, v) in headers.iter() {
    let kn = k.as_str().to_ascii_lowercase();
    // 剔除列表
    if kn == "host"
        || kn == "content-length"
        || kn == "accept-encoding"
        || kn == "authorization"
        || kn == "x-api-key"
        || kn == resolver::ENDPOINT_HEADER      // x-ccmomo-endpoint
        || kn == resolver::ENDPOINT_HEADER_ALT   // x-endpoint-name
        || (override_ua && kn == "user-agent")
    {
        continue;
    }
    // 其他头部原样转发
    rb = rb.header(k.as_str(), val);
}
```

**结论**：以下头部已经自动透传（不在剔除列表中）：
- session_id
- x-session-id
- conversation_id
- x-claude-code-session-id
- claude-code-session-id
- metadata（作为头部字段）
- originator

### 2.2 第二步：分析请求体处理逻辑

文件：`src-tauri/src/modules/proxy/forward.rs:309-336`

```rust
let attempt_body: Bytes = if needs_transform {
    // Claude → OpenAI 转换：重新序列化
    match &body_json {
        Some(cj) => get_transformer(format)
            .transform_request(cj, Some(&ep.model))
            .ok()
            .and_then(|v| serde_json::to_vec(&v).ok())
            .map(Bytes::from)
            .unwrap_or_else(|| body.clone()),
        None => body.clone(),
    }
} else if !ep.model.is_empty() {
    // model 锁定：只覆盖 model，其他原样
    match &body_json {
        Some(cj) => {
            let mut v = cj.clone();
            if let Some(o) = v.as_object_mut() {
                o.insert("model".to_string(), Value::String(ep.model.clone()));
            }
            serde_json::to_vec(&v).map(Bytes::from).unwrap_or_else(|_| body.clone())
        }
        None => body.clone(),
    }
} else {
    body.clone()  // 原样转发
};
```

**结论**：
- 直通场景（Claude→Claude、OpenAI→OpenAI）：body.clone() 原样转发，metadata 不丢失
- model 锁定场景：只覆盖 model 字段，metadata 不丢失
- Claude→OpenAI 转换场景：调用 transform_request，**需要检查是否保留 metadata**

### 2.3 第三步：分析 Claude→OpenAI 转换函数

文件：`src-tauri/src/modules/transform/claude_openai.rs:20-98`


**发现问题**：Claude→OpenAI 转换时，metadata 字段被丢弃。

## 三、问题分析

### 3.1 影响范围

| 转换场景 | metadata 是否丢失 | 影响 |
|---------|------------------|------|
| Claude → Claude 直通 | 不丢失 | body.clone() 原样转发 |
| Claude → OpenAI 转换 | **丢失** | session_id 等信息丢失 |
| OpenAI → OpenAI 直通 | 不丢失 | body.clone() 原样转发 |

### 3.2 受影响的字段

metadata 中可能包含的重要字段：

| 字段 | 来源 | 用途 |
|------|------|------|
| metadata.user_id | Claude Code | 会话标识（格式：user_xxx_session_yyy） |
| metadata.session_id | Claude Code | 会话标识 |
| metadata.agent_id | Claude Code | 子代理标识 |
| metadata.agent_type | Claude Code | 子代理类型 |

### 3.3 受影响的下游系统

如果 tauri-gateway 后接以下系统，会受到影响：

1. **sub2api**：依赖 metadata.user_id/metadata.session_id 进行会话固定
2. **cc-switch**：依赖 metadata.session_id 进行日志追踪
3. **其他需要会话标识的代理系统**

## 四、推断过程

```
问题：转发时会出错吗？

推断链：
1. 头部透传 → session_id 等头部已自动透传 ✓
2. 请求体处理 → 直通场景原样转发 ✓
3. 请求体处理 → Claude→OpenAI 转换场景需要验证
4. 查看 claude_request_to_openai 函数 → 发现只转换了特定字段
5. 确认：metadata 字段在转换时被丢弃 ✗

结论：Claude→OpenAI 转换场景会丢失 metadata，可能导致下游会话管理失败。
```

---

## 五、修复建议

### 5.1 方案一：保留 metadata 字段（推荐）

在 `claude_request_to_openai` 函数中添加 metadata 保留逻辑：

```rust
pub fn claude_request_to_openai(claude: &Value, endpoint_model: Option<&str>) -> Value {
    let mut out = serde_json::Map::new();

    // ... 现有转换逻辑 ...

    // 保留 metadata 字段（用于 session_id 等会话标识）
    if let Some(metadata) = claude.get("metadata") {
        if !metadata.is_null() {
            out.insert("metadata".into(), metadata.clone());
        }
    }

    Value::Object(out)
}
```

**优点**：
- 改动最小，只添加 3 行代码
- 保持向后兼容
- 不影响现有功能

**缺点**：
- OpenAI API 可能不识别 metadata 字段（但通常会忽略未知字段）

### 5.2 方案二：提取关键字段到顶层

将 metadata 中的关键字段提取到请求体顶层：

```rust
// 提取 session_id 到顶层
if let Some(user_id) = claude.get("metadata").and_then(|m| m.get("user_id")) {
    out.insert("user_id".into(), user_id.clone());
}
```

**优点**：
- 符合 OpenAI API 规范

**缺点**：
- 需要了解下游系统期望的字段格式
- 可能丢失其他 metadata 信息

### 5.3 建议

采用**方案一**，原因：
1. 改动最小，风险最低
2. 保留完整信息，下游系统可自行提取
3. OpenAI API 对未知字段通常忽略，不会报错

```rust
fn test_claude_to_openai_preserves_metadata() {
```
